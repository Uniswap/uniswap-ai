# Liquidity Position Correctness Rubric

Evaluate whether the response correctly plans a Uniswap liquidity position.

## Required Elements (Must Include)

Score based on how many of these are correctly implemented:

1. **Token Pair Verification**
   - Correctly identifies both tokens in the pair
   - Verifies or uses correct contract addresses for the target chain

2. **Pool Version Awareness**
   - Correctly handles the requested Uniswap version (V2, V3, or V4)
   - Applies version-appropriate parameters (price range for V3/V4, full range for V2)

3. **Price Range / Fee Tier**
   - Suggests an appropriate price range (for V3/V4) or explains full range (for V2)
   - Recommends or discusses fee tier selection
   - Explains trade-offs of the chosen parameters

4. **Deep Link Generation**
   - Generates a valid Uniswap deep link (app.uniswap.org)
   - Includes correct parameters (tokens, chain, version, fee tier)

5. **Position Economics**
   - Explains relevant risks (impermanent loss, out-of-range positions)
   - Provides context on current pool state (price, liquidity depth, volume)

## Scoring Guide

- 5/5 elements correct: 1.0
- 4/5 elements correct: 0.85
- 3/5 elements correct: 0.7
- 2/5 elements correct: 0.5
- 1/5 elements correct: 0.25
- 0/5 elements correct: 0.0

## Instructions

Evaluate the response for technical accuracy in planning the liquidity position. The response should demonstrate understanding of the specific Uniswap version's mechanics and generate a properly formatted deep link.
