# Completeness Rubric for lp-integration

Evaluate whether the generated output includes all expected elements for a Uniswap LP integration.

## Expected Elements (Should Include)

Score based on how many of these are present:

1. **Documentation**
   - Comments explaining the approval -> action flow
   - Notes about common pitfalls (inner-transaction signing, response field names, v2 fee restrictions)

2. **Error Handling**
   - Checks res.ok before calling res.json()
   - Validates LP transaction before broadcasting (non-empty data, valid addresses)
   - Handles token approval failures and API errors with status codes

3. **Best Practices**
   - Amounts are wei-denominated integer strings (uses parseUnits or equivalent, never raw decimals)
   - Validates transaction data is non-empty and starts with 0x before broadcasting
   - Permit fields sent as a matched pair (batchPermitData + signature both present, or both absent)
   - User confirmation (AskUserQuestion or equivalent) before any gas-spending transaction

4. **Maintainability**
   - LP_API_BASE_URL extracted as a named constant (not inline string literals)
   - API key loaded from environment variable, not hardcoded
   - Logic is organized into clear helper functions (e.g. lpFetch, validateLpTransaction)

5. **Completeness**
   - Includes all necessary imports
   - Demonstrates the full lifecycle appropriate to the request (approval check through broadcast)
   - Displays adjustedMinPrice / adjustedMaxPrice to the user (not original priceBounds input)

## Penalties (Should Not Include)

Deduct points for:
- Signing the outer approval wrapper instead of approval.transaction (-0.15)
- Using `approvals` instead of `transactions` from check_approval response (-0.15)
- Using `minPrice` / `maxPrice` instead of `adjustedMinPrice` / `adjustedMaxPrice` (-0.10)
- Missing pre-broadcast validation of transaction data (-0.10)
- Hardcoding API key in generated code (-0.15)

## Scoring Guide

Base score from expected elements:
- 5/5 elements: 1.0
- 4/5 elements: 0.85
- 3/5 elements: 0.65
- 2/5 elements: 0.5
- 1/5 elements: 0.3
- 0/5 elements: 0.0

Then subtract any penalties.

## Instructions

Count the expected elements present, then apply any penalties for issues found. Provide the final score.
