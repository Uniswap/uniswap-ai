# Correctness Rubric for copy-trade

Evaluate whether the generated output correctly mirrors a leader wallet within guardrails, using a cursor and RPC polling, while delegating execution.

## Required Elements (Must Include)

Score based on how many of these are correctly implemented:

1. **Delegation to swap-integration (no reimplementation)**
   - Every mirror swap is delegated to the swap-integration Trading API flow (`check_approval`, then `quote`, then `swap`) with signing via viem-integration.
   - Verify the response does NOT hand-roll quoting, approvals, swap calldata, routing, or signing.

2. **Correct chain and address usage from references**
   - Uses the target chain id from the case input (the case targets Robinhood Chain, `4663`) and reads contract / token addresses from that chain's config (`references/robinhood-chain.md`) rather than hardcoding.
   - Verify the leader address and allowlist tokens are resolved against the references and no addresses are invented.

3. **Execution mode plus guardrails**
   - Supports `executionMode` of `confirm` (default) and `autonomous`, and applies the mirror guardrails: chain filter (4663 only), asset allowlist match, and per-mirror position-size cap, plus the follower's portfolio state.
   - Verify autonomous mode is also gated by spend cap, token allowlist, dry-run first, and a kill switch, and that failing actions are skipped.

4. **State, cursor, and idempotency handling**
   - Reads the stored cursor (last processed block or log position) and the already-mirrored action set, writes state immediately after each action resolves (broadcast confirmed, recording the follower tx hash, or skip recorded with its reason) rather than batching to run-exit, and advances the durable cursor only past committed actions so no action is mirrored twice even across a mid-run crash.
   - Verify the run is deterministic and self-contained per wake.

5. **RPC polling and input validation**
   - v1 reads the leader's new actions by polling the RPC and scanning logs since the cursor (v1 reads via RPC polling and log scanning).
   - Verify inputs are validated (leader address, allowlist tokens resolve on 4663) before any execution.

## Scoring Guide

- 5/5 elements correct: 1.0
- 4/5 elements correct: 0.8
- 3/5 elements correct: 0.6
- 2/5 elements correct: 0.4
- 1/5 elements correct: 0.2
- 0/5 elements correct: 0.0

## Instructions

Count how many of the required elements are correctly implemented and provide the corresponding score.
