# Correctness Rubric for index-bot

Evaluate whether the generated output correctly builds a weighted basket, buys it in one pass, and rebalances on drift while delegating execution.

## Required Elements (Must Include)

Score based on how many of these are correctly implemented:

1. **Delegation to swap-integration (no reimplementation)**
   - Every buy and rebalance trade is delegated to the swap-integration Trading API flow (`check_approval`, then `quote`, then `swap`) with signing via viem-integration; any LP goes through the v4 SDK integration.
   - Verify the response does NOT hand-roll quoting, approvals, swap calldata, routing, signing, or LP transactions.

2. **Correct chain and address usage from references**
   - Uses the target chain id from the case input (the case targets Robinhood Chain, `4663`) and reads contract / token addresses (including v4 PoolManager / PositionManager if LP is used) from that chain's config (`references/robinhood-chain.md`) rather than hardcoding.
   - Verify each ticker is resolved via the references and no addresses are invented.

3. **Execution mode plus guardrails**
   - Supports `executionMode` of `confirm` (default) and `autonomous`.
   - Verify autonomous mode is gated by all guardrails: spend cap (per run and per period), token allowlist, dry-run first, and a kill switch.

4. **State and idempotency handling (weights and rebalance)**
   - Records target weights and the last rebalance in JSON state, reads them to compute drift, and writes state only after successful broadcasts.
   - Verify a rebalance wake trades only the legs that exceed the drift threshold rather than re-buying the whole basket.

5. **Basket parsing, sizing, and input validation**
   - Parses the basket spec and computes per-leg sizing from target weights and funding, buying all legs in one pass.
   - Verify inputs are validated (each ticker resolves on the case's target chain (here `4663`), weights sum to 100%) before any execution.

## Scoring Guide

- 5/5 elements correct: 1.0
- 4/5 elements correct: 0.8
- 3/5 elements correct: 0.6
- 2/5 elements correct: 0.4
- 1/5 elements correct: 0.2
- 0/5 elements correct: 0.0

## Instructions

Count how many of the required elements are correctly implemented and provide the corresponding score.
