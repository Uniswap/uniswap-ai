# Completeness Rubric for index-bot

Evaluate whether the generated output includes all expected elements and follows the plugin's conventions.

## Expected Elements (Should Include)

Score based on how many of these are present:

1. **Prerequisites referenced**
   - References the swap-integration, viem-integration, and (for LP) v4 SDK integration prerequisites as the execution path.
   - Points to `references/robinhood-chain.md` and `references/execution-model.md` for config and rules.

2. **Compliance handling**
   - Handles token-level transfer-restriction reverts gracefully and reports them per leg.
   - Respects equity market hours and surfaces the repo root `DISCLAIMER.md`.

3. **Skill-specific steps (index)**
   - Covers basket-spec parsing, per-leg sizing from target weights, the one-pass buy, and weekly rebalance on drift.
   - Describes computing drift from recorded target weights and trading only drifted legs.

4. **Execution and guardrails framing**
   - States the execution mode (confirm default vs autonomous) and the autonomous guardrails (spend cap, allowlist, dry-run, kill switch).

## Penalties (Should Not Include)

Deduct points for:
- Hand-rolled quote, approval, swap, signing, or LP transaction logic (-0.15)
- Hardcoded addresses or chain values instead of reading references (-0.1)
- Re-buying the whole basket on rebalance instead of trading only drifted legs (-0.1)

## Scoring Guide

Base score from expected elements:
- 4/4 elements: 1.0
- 3/4 elements: 0.8
- 2/4 elements: 0.55
- 1/4 elements: 0.3
- 0/4 elements: 0.0

Then subtract any penalties.

## Instructions

Count the expected elements present, then apply any penalties for issues found. Provide the final score.
