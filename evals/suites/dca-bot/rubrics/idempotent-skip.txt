# Idempotent Skip Rubric for dca-bot

Evaluate whether the response correctly skips a buy when the current period has
already been bought, with no double spend.

Background: the state file shows `lastBuyPeriod: "2026-06-24"`. The current wake is
`2026-06-24T18:00:00Z`, so the current UTC-day period (`2026-06-24`) has already been
bought. The bot buys at most once per period.

## Required Behaviors

Score based on how many of these are correctly handled:

1. **Reads state and computes the period**
   - The response reads the JSON state file and computes the current period
     (`2026-06-24`) from the daily cadence.

2. **Detects the already-bought period and skips**
   - The response compares the current period against `lastBuyPeriod`, finds they match,
     and skips the buy. It treats this as a normal, clean outcome (not an error).

3. **No double spend, no state corruption**
   - The response does NOT broadcast a second buy for today.
   - It does NOT overwrite or clobber the recorded buy / `lastBuyPeriod`.

## Scoring Guide

- 3/3 behaviors correct: 1.0
- 2/3 behaviors correct: 0.7
- 1/3 behaviors correct: 0.4
- 0/3 behaviors correct: 0.0

A response that broadcasts a second buy for the already-bought period scores 0.0
regardless of other behaviors.

## Instructions

Count how many behaviors are correctly handled and provide the corresponding score.
