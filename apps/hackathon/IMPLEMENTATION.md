# Hackathon Website - Implementation Summary

## Overview

MVP website for the Uniswap AI Hackathon, built as a React SPA within the `uniswap-ai` monorepo. The site showcases hackathon info and project submissions sourced from GitHub Issues. Designed with Uniswap's Spore design system tokens. No auth required; everything is public.

## Tech Stack

| Concern    | Choice                                              |
| ---------- | --------------------------------------------------- |
| Framework  | React 19 + TypeScript                               |
| Build      | Vite 6                                              |
| Routing    | React Router v7 (client-side SPA with data loaders) |
| Styling    | CSS Modules + CSS custom properties (Spore tokens)  |
| GitHub API | `@octokit/rest`                                     |
| Markdown   | `react-markdown` + `remark-gfm`                     |
| Font       | Inter (Google Fonts)                                |
| Deployment | Vercel (to be configured)                           |

## Pages

1. **Landing** (`/`) - Hero with countdown timer, about section, prize tiers, how-to-participate steps, FAQ accordion
2. **Projects Listing** (`/projects`) - 2-column grid of project cards fetched from GitHub Issues
3. **Project Detail** (`/projects/:slug`) - Full project info with markdown description, links, team members, timestamps
4. **Dashboard** (`/dashboard`) - 3 metric cards (countdown, submission count, participant count) + activity feed

## Data Flow

- GitHub Issues labeled `hackathon-submission` are the source of truth
- `lib/github.ts` uses `@octokit/rest` to fetch issues (list endpoint returns full bodies)
- `lib/parseSubmission.ts` parses `### Heading` sections (generated by the YAML form template) into typed `HackathonProject` objects
- Slug format: `{kebab-case-name}-{issueNumber}` (e.g., `my-project-42`)
- React Router loaders fetch data with a 1-minute client-side cache to reduce API calls
- Optional `VITE_GITHUB_TOKEN` env var for higher rate limits (60/hr unauth vs 5000/hr with token)

## Spore Design Tokens

Extracted from Uniswap's interface repo, defined as CSS custom properties:

- Surface colors: `--color-surface1` (#131313), `--color-surface2` (#1F1F1F), `--color-surface3`
- Text colors: `--color-neutral1` (white), `--color-neutral2` (65% white), `--color-neutral3` (38% white)
- Accent: `--color-accent1` (#FF37C7 - Uniswap pink)
- Status: success (#21C95E), warning (#FFBF17), critical (#FF593C)
- Border radii: sm (4px) through full (9999px)
- Spacing scale: 4px increments

## Vercel Deployment Setup

To deploy, create a new Vercel project with:

- **Root directory**: `apps/hackathon`
- **Framework preset**: Vite
- **Build command**: `cd ../.. && npx nx run hackathon:build`
- **Output directory**: `dist`
- **Install command**: `cd ../.. && npm ci`
- **Environment variable** (optional): `VITE_GITHUB_TOKEN` for higher GitHub API rate limits

The `vercel.json` SPA rewrite ensures client-side routing works for direct URL access.
